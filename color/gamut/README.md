# gamut

`color.gamut` provides display-primary gamut feasibility tests, Lab/LCHab
boundary construction, xy/Lab coverage metrics, Pointer real-surface gamut,
MacAdam optimal-colour limits, and high-level gamut summaries.

All public `XYZ` values use the project convention: `Y=100`.

## Scope

This module works with linear colour stimuli and geometric gamut boundaries.
It does not perform RGB transfer functions, chromatic adaptation, display
calibration, colour appearance modelling, or spectral integration for ordinary
colour conversion.

The module currently covers:

- display-primary gamuts, including three-primary and multi-primary displays;
- regular `L* x hue` Lab/LCHab boundary objects;
- xy area coverage and Lab volume coverage;
- Pointer real-surface colour gamut;
- MacAdam optimal-colour limits, using cached A/C/D65 tables or computed
  limits for custom CMFs/illuminants;
- a high-level `analyze_gamut(...)` summary.

## Display Primaries

```python
from color.gamut import DisplayPrimaries

primaries = DisplayPrimaries.from_RGB_colourspace("sRGB")
```

`DisplayPrimaries.primaries_XYZ` has shape `(n, 3)`. Each row is the linear
`XYZ(Y=100)` stimulus of one primary, not xy chromaticity.

For RGB colourspaces, the primaries are taken from the registered linear
`RGB -> XYZ` matrix:

```python
srgb = DisplayPrimaries.from_RGB_colourspace("sRGB")
rec2020 = DisplayPrimaries.from_RGB_colourspace("Rec.2020")
```

## Primary Feasibility

```python
from color.gamut import is_within_primary_gamut

inside = is_within_primary_gamut([20, 30, 40], srgb)
```

`method="auto"` uses:

- `matrix` for exactly three primaries;
- convex-hull halfspaces for four or more primaries.

The multi-primary inside test is vectorised. It answers whether an `XYZ` value
is within the convex volume generated by all off/on primary combinations.

## Primary Weights

```python
from color.gamut import solve_primary_weights

weights = solve_primary_weights([20, 30, 40], srgb)
```

For three primaries, the solution is unique. For four or more primaries, the
solution is usually non-unique. v1 uses `scipy.optimize.linprog` to return one
feasible solution with `objective="min_sum"`.

This is only a feasibility solver. It is not a full display-driving model and
does not choose perceptually or electrically optimal multi-primary weights.

## LCHab Boundary

```python
from color.gamut import compute_LCH_gamut_boundary

boundary = compute_LCH_gamut_boundary(
    "Rec.2020",
    L_values=range(0, 101, 5),
    hue_values=range(0, 361, 5),
    C_upper=260,
)
```

`GamutBoundary.C_max` stores the maximum reachable `C*` for every requested
`(L*, hue)` pair:

```text
C_max.shape == (len(L_values), len(hue_values))
```

Useful boundary methods:

```python
boundary.to_LCHab()
boundary.to_Lab()
boundary.to_XYZ()
boundary.slice_L(50)

boundary.area_at_L(50)
boundary.lab_volume()
boundary.gamut_rings([25, 50, 75, 100])
boundary.ring_area()
```

## xy Boundary vs Lab Projection

Use `xy_boundary()` when you need the preferred CIE xy-plane boundary:

```python
xy = boundary.xy_boundary()
```

For display primaries, this returns the primary xy convex hull. Pointer and
MacAdam boundary subclasses override it with their appropriate xy boundary.
Generic boundaries without a specialised implementation fall back to the
convex hull of sampled boundary XYZ projected into xy.

Use these methods only for Lab `a*b*` plane summaries:

```python
boundary.projected_chroma_boundary()
boundary.projected_lightness_boundary()
boundary.projected_LCHab_boundary()
boundary.projected_ab_boundary()
boundary.projected_ab_area()
```

Do not convert `projected_LCHab_boundary()` or `projected_ab_boundary()` into
xy and interpret that result as an xy gamut boundary. The Lab/LCHab to xy
mapping is nonlinear and loses luminance information.

## xy Coverage

Coverage is directional:

```text
coverage(test, reference) = intersection(test, reference) / area(reference)
```

For RGB names, `DisplayPrimaries`, or primary `XYZ` rows:

```python
from color.gamut import xy_gamut_coverage

xy_gamut_coverage("sRGB", "Rec.2020")
xy_gamut_coverage(rec2020, srgb)
xy_gamut_coverage(srgb.primaries_XYZ, rec2020.primaries_XYZ)
```

For already available xy points or xy hulls:

```python
from color.gamut import pointer_gamut_published_xy_boundary, xy_gamut_coverage_from_xy

rec2020_xy = [
    [0.708, 0.292],
    [0.170, 0.797],
    [0.131, 0.046],
]
pointer_xy = pointer_gamut_published_xy_boundary()

coverage = xy_gamut_coverage_from_xy(rec2020_xy, pointer_xy)
```

The `*_from_xy` helpers compute a convex hull from the input points. Three,
four, and higher-primary xy point sets can be passed directly.

## Lab Coverage

Lab coverage compares two `GamutBoundary` objects:

```python
from color.gamut import lab_gamut_coverage

coverage = lab_gamut_coverage(test_boundary, reference_boundary)
```

It is also directional:

```text
coverage(test, reference) = overlap_volume(test, reference) / volume(reference)
```

If the two boundaries use different `whitepoint_XYZ`, a `UserWarning` is
emitted and no chromatic adaptation is performed. If the L/hue grids differ,
a `UserWarning` is emitted and the test boundary is interpolated onto the
reference grid.

## Gamut Analysis

`analyze_gamut(...)` collects the common metrics in one result object:

```python
from color.gamut import analyze_gamut

analysis = analyze_gamut("sRGB")

analysis.xy_area
analysis.lab_volume
analysis.xy_coverage_pointer
analysis.volume_coverage_macadam_d65
analysis.warnings
```

Supported input:

```python
analyze_gamut("sRGB")
analyze_gamut(DisplayPrimaries(...))
analyze_gamut(existing_boundary)
```

The result compares the current gamut against three references:

- Rec.2020;
- Pointer real-surface gamut;
- D65 MacAdam optimal-colour limits.

The first version does not compute visible-locus / horseshoe coverage.

## Pointer Gamut

Pointer's gamut is an empirical real-surface colour gamut, not a display
primary gamut:

```python
from color.gamut import (
    is_within_pointer_gamut,
    pointer_gamut,
    pointer_gamut_published_xy_boundary,
)

pointer = pointer_gamut()
pointer_xy = pointer_gamut_published_xy_boundary()
inside = is_within_pointer_gamut([32.05, 41.31, 51.00])
```

`pointer_gamut()` returns a `PointerGamutBoundary`. Its `xy_boundary()` is the
published 32-point Pointer xy boundary, not a projection of the Lab/LCHab
boundary.

## MacAdam Limits

MacAdam limits describe optimal colour stimuli for an illuminant.

Use the unified entry point:

```python
from color.gamut import (
    is_within_macadam_limits,
    macadam_limits,
    macadam_limits_published_xy_boundary,
)

boundary = macadam_limits("D65")
inside = is_within_macadam_limits([39.57, 51.0, 32.89], "A")
xy = macadam_limits_published_xy_boundary("D65")
```

`macadam_limits(..., source="auto")` is the default:

- plain `A`, `C`, and `D65` requests use cached tables in
  `color/data/gamut_data`;
- custom CMFs, custom illuminants, or a custom spectral shape switch to the
  computed route.

Force the computed route when needed:

```python
from color.spectra import SpectralShape

computed = macadam_limits(
    "D65",
    source="computed",
    shape=SpectralShape(400, 700, 5),
    L_values=range(0, 101, 10),
)
```

The computed route follows the brightness-factor method:

```text
L* -> R = Y / Yn -> Type 1 / Type 2 rectangular reflectance spectra
   -> spectral integration -> XYZ -> Lab/LCHab boundary
```

`R` is derived from `L*`; it is not an independent input.

Advanced computed helpers such as `computed_macadam_limits_data(...)` and
`computed_macadam_limits_XYZ(...)` remain available from `color.gamut.macadam`,
but they are not top-level `color.gamut` APIs.

Visible spectrum volume is not implemented here. It needs a separate energy,
luminance, or SPD constraint before it is a well-defined volume.
